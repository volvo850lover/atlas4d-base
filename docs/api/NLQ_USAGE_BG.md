# NLQ Ръководство (Natural Language Query)

Atlas4D поддържа заявки на естествен език на български и английски.

## Какво е NLQ?

NLQ (Natural Language Query) позволява да питате Atlas4D с обикновени изречения вместо SQL или API заявки.

**Примери:**
- "Покажи всички наблюдения от последния час"
- "Колко аномалии има днес?"
- "Какво е времето в Бургас?"

## Как да използвам NLQ?

### През уеб интерфейса

1. Отворете https://atlas4d.tech (или вашата инсталация)
2. Кликнете на **NLQ** в навигацията
3. Въведете въпроса си на български или английски
4. Натиснете Enter или бутона за изпращане

### През Python SDK
```python
from atlas4d import Client

client = Client(host="localhost", port=8081)

# Въпрос на български
answer = client.ask("Какво е Atlas4D?", lang="bg")
print(answer.text)

# Въпрос на английски
answer = client.ask("How many observations today?", lang="en")
print(answer.text)
```

### През REST API
```bash
curl -X POST "http://localhost:8081/api/nlq/rag" \
  -H "Content-Type: application/json" \
  -d '{"q": "Какво е Atlas4D?", "lang": "bg", "top_k": 3}'
```

## Типове заявки

### Документация (RAG)
Въпроси за Atlas4D платформата:
- "Какво е Atlas4D?"
- "Как да инсталирам платформата?"
- "Какви модули има?"
- "Как да създам нов модул?"

### Данни
Въпроси за съхранените данни:
- "Покажи последните 10 наблюдения"
- "Колко аномалии има от вчера?"
- "Какви са източниците на данни?"

### Време и място
Въпроси с географски и времеви контекст:
- "Какво е времето в София?"
- "Покажи наблюдения около Бургас"
- "Какво се случи през последния час?"

## Примери на български

| Въпрос | Какво прави |
|--------|-------------|
| Какво е Atlas4D? | Обяснява платформата |
| Как да инсталирам? | Показва стъпките за инсталация |
| Какви модули има? | Изброява наличните модули |
| Покажи последните данни | Връща скорошни наблюдения |
| Колко аномалии днес? | Брои аномалиите |

## Съвети за по-добри резултати

1. **Бъдете конкретни** - "Покажи GPS наблюдения от Бургас" вместо "Покажи данни"

2. **Използвайте времеви рамки** - "от последния час", "днес", "тази седмица"

3. **Посочете място** - "в София", "около Бургас", "в радиус 10км"

4. **Един въпрос наведнъж** - Избягвайте сложни комбинирани въпроси

## Ограничения

- NLQ работи най-добре с кратки, ясни въпроси
- Сложни анализи изискват STSQL или директни API заявки
- Някои специфични термини може да не се разпознаят
